01 — Setup

Configuración inicial

Pasos para levantar el sistema por primera vez.

1

Base de datos MySQL

Crear inventory_db y configurar credenciales en backend/.env

2

Instalar dependencias

cd backend && npm install
cd frontend && npm install

3

Correr el seeder

npm run seed — carga BOTANAS HEBI, 16 familias y ~150 productos.

4

Iniciar servidores

Backend: npm run dev (4000)
Frontend: npm run dev (5173)

5

Primer acceso

admin@botanashebi.com
Admin1234!

6

Configurar productos

Asignar productType: materia_prima / empaque / granel / terminado.

03 — Ciclo completo

Flujo operativo

De la compra de materia prima hasta el embarque al cliente.

sequenceDiagram actor U as Usuario participant FE as Frontend participant API as Backend participant DB as MySQL Note over U,DB: Compra de Materia Prima U->>FE: Movimientos → + Registrar (Entrada) FE->>API: POST /api/inventory/movements API->>DB: stock += qty · INSERT InventoryMovement DB-->>FE: OK Note over U,DB: Producción — Cocina U->>FE: Producción → + Nueva orden (Cocina) FE->>API: POST /api/production API->>DB: INSERT ProductionOrder + Items U->>FE: Completar y afectar inventario FE->>API: PATCH /api/production/:id/complete API->>DB: stock -= inputs · stock += outputs · INSERT Movements DB-->>FE: Completado Note over U,DB: Producción — Envasado U->>FE: Producción → + Nueva orden (Envasado) FE->>API: POST /api/production + PATCH .../complete API->>DB: stock -= granel/empaque · stock += terminado DB-->>FE: Completado Note over U,DB: Venta y Embarque U->>FE: Notas de Venta → + Nueva nota FE->>API: POST /api/sales API->>DB: INSERT SaleOrder + Items (borrador) U->>FE: Embarcar FE->>API: PATCH /api/sales/:id/ship API->>DB: stock -= qty terminado · INSERT Movements DB-->>FE: Embarcado
04 — Módulo

Órdenes de producción

Estados y transiciones de una Orden de Producción.

stateDiagram-v2 [*] --> Borrador : Crear (POST /production) Borrador --> EnProceso : Iniciar (PATCH .../status) Borrador --> Completado : Completar directo EnProceso --> Completado : Completar (PATCH .../complete) Borrador --> Cancelado : Cancelar EnProceso --> Cancelado : Cancelar Completado --> [*] Cancelado --> [*]
Al completar (transacción): inputs → salida de inventario · outputs → entrada de inventario. Requiere stock suficiente en todos los insumos.
05 — Módulo

Notas de venta

Estados y acciones de una Nota de Venta.

stateDiagram-v2 [*] --> Borrador : Crear (POST /sales) Borrador --> Confirmada : Confirmar (PATCH .../confirm) Borrador --> Embarcada : Embarcar directo Confirmada --> Embarcada : Embarcar (PATCH .../ship) Borrador --> Cancelada : Cancelar Confirmada --> Cancelada : Cancelar Embarcada --> [*] Cancelada --> [*]
Al embarcar (transacción): descuenta stock de producto terminado y registra movimientos de salida.
06 — Módulo

Movimientos de inventario

Todo cambio de stock queda registrado como movimiento.

flowchart TD O["Origen del movimiento"] O --> A["Entrada\nCompra MP · OP output"] O --> B["Salida\nOP input · Embarque · Merma"] O --> C["Ajuste\nCorreción manual"] A --> D["stock = stock + qty"] B --> E["stock = stock - qty\n(valida stock mayor o igual qty)"] C --> F["stock = valor capturado"] D --> G["INSERT InventoryMovement"] E --> G F --> G
07 — Seguridad

Autenticación JWT

Token Bearer en cada request. El frontend redirige al login si el token expira.

sequenceDiagram actor U as Usuario participant FE as Frontend participant API as Backend U->>FE: Email + contraseña FE->>API: POST /api/auth/login API->>API: bcrypt.compare() alt Válido API-->>FE: token + user FE->>FE: localStorage token FE-->>U: Dashboard else Inválido API-->>FE: 401 FE-->>U: Error end Note over FE,API: Cada request autenticado FE->>API: GET /api/products Authorization Bearer token API->>API: jwt.verify() User.findByPk() alt Token válido API-->>FE: 200 + datos else Expirado o inválido API-->>FE: 401 FE->>FE: localStorage.clear() redirige a login end
08 — Accesos

Roles y permisos

Verificados en el middleware authorize() del backend.

👑 Admin

  • Todo lo del sistema
  • Crear y gestionar usuarios
  • Desactivar productos y familias
  • Embarcar notas de venta
  • Ver valuación de inventario

🧑‍💼 Manager

  • Crear y editar productos
  • Crear y editar familias y proveedores
  • Completar órdenes de producción
  • Embarcar notas de venta
  • Ver valuación de inventario

👷 Empleado

  • Ver productos y stock
  • Registrar movimientos manuales
  • Crear y confirmar notas de venta
  • Ver historial y reportes básicos
09 — Referencia

Estados de los documentos

Orden de Producción

Borrador→ En proceso→ Completado ✓
Borrador · En proceso→ Cancelado

Nota de Venta

Borrador→ Confirmada→ Embarcada
Borrador · Confirmada→ Cancelada

Producto

A — Activo→ I — Inactivo

Soft delete: no se elimina de la base de datos.

Tipos de Producto

materia_prima material_empaque granel terminado general

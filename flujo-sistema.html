🚀 Configuración inicial

Pasos para levantar el sistema por primera vez.

1

Crear base de datos MySQL

Crear la BD inventory_db en MySQL. Configurar credenciales en backend/.env

2

Instalar dependencias

cd backend && npm install
cd frontend && npm install

3

Correr el seeder

npm run seed — carga BOTANAS HEBI, 16 familias, ~150 productos y el usuario admin.

4

Iniciar servidores

Backend: npm run dev (puerto 4000)
Frontend: npm run dev (puerto 5173)

5

Primer acceso

Email: admin@botanashebi.com
Password: Admin1234!

6

Configurar productos

Asignar productType a cada producto: materia_prima / material_empaque / granel / terminado

Ciclo de vida de un lote desde la compra de materia prima hasta la venta.

⚙️ Detalle: Módulo de Producción

Estados y transiciones de una Orden de Producción.

stateDiagram-v2 [*] --> Borrador : Crear orden\n(POST /production) Borrador --> EnProceso : Iniciar producción\n(PATCH .../status) Borrador --> Completado : Completar directo\n(PATCH .../complete) EnProceso --> Completado : Completar\n✓ Afecta inventario\n(PATCH .../complete) Borrador --> Cancelado : Cancelar EnProceso --> Cancelado : Cancelar Completado --> [*] Cancelado --> [*] note right of Completado Al completar (transacción): - items input → salida de inventario - items output → entrada de inventario - Registra movimientos automáticos end note
⚠️ Requisito para completar una orden Los productos definidos como insumo (input) deben tener stock suficiente. Si alguno tiene stock = 0, la operación falla con error 400 "Stock insuficiente". Registra primero una entrada de inventario para esos productos.

🔐 Autenticación y rutas protegidas

JWT en ambos lados: el backend valida el token en cada request, el frontend redirige al login si no existe.

sequenceDiagram actor U as Usuario participant FE as Frontend participant API as Backend U->>FE: Ingresa email + contraseña FE->>API: POST /api/auth/login API->>API: bcrypt.compare(password, hash) alt Credenciales válidas API-->>FE: { token, user } FE->>FE: localStorage.setItem("token") FE-->>U: Redirige al Dashboard else Inválidas API-->>FE: 401 "Credenciales inválidas" FE-->>U: Muestra error end Note over FE,API: Cada request autenticado FE->>API: GET /api/products\nAuthorization: Bearer {token} API->>API: jwt.verify(token)\nUser.findByPk(decoded.id) alt Token válido API-->>FE: 200 + datos else Token expirado / inválido API-->>FE: 401 FE->>FE: Limpia localStorage\nRedirige a /login end

👥 Roles y permisos

Tres niveles de acceso. El rol se verifica en el middleware authorize() del backend.

👑 Admin

  • Todo lo de manager
  • Registrar usuarios
  • Desactivar productos
  • Desactivar categorías / proveedores
  • Embarcar notas de venta
  • Ver valuación de inventario

🧑‍💼 Manager

  • Todo lo de employee
  • Crear / editar productos
  • Crear / editar categorías
  • Crear / editar proveedores
  • Crear / completar órdenes de producción
  • Embarcar notas de venta
  • Ver valuación de inventario

👷 Employee

  • Ver productos y stock
  • Registrar movimientos de inventario
  • Crear notas de venta (borrador)
  • Confirmar notas de venta
  • Ver historial de movimientos
  • Ver reportes básicos

🏷️ Estados de los documentos

Orden de Producción

Borrador → En Proceso → Completado ✓
Borrador / En Proceso → Cancelado

Nota de Venta

Borrador → Confirmada → Embarcada 🚚
Borrador / Confirmada → Cancelada

Producto

A — Activo → I — Inactivo

Inactivo = soft delete (no se elimina de la BD)

Tipos de Producto

materia_prima material_empaque granel terminado general
